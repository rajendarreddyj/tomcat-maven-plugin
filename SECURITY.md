# Security Policy

## Supported Versions
We support and patch the latest released version. Pre-release snapshots (`-SNAPSHOT`) are not guaranteed to receive security fixes.

## Reporting a Vulnerability
Please report security issues privately. Do NOT create a public issue.

- Email: rajendarreddyj@gmail.com
- Include a clear description, reproduction steps, affected version, and impact
- Optionally include a minimal project or POM to reproduce

We will acknowledge receipt within 3 business days and aim to provide an initial assessment within 7 business days.

## Disclosure
We follow responsible disclosure. Once a fix is available, we will publish release notes referencing the issue, and coordinate timelines when needed.

## Dependencies
We rely on Maven Central artifacts and keep dependencies updated. If you discover a vulnerable transitive dependency, please report it with coordinates and suggested remediation.
